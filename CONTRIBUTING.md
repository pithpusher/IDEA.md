# Contributing to IDEA.md

Thanks for your interest in improving the IDEA.md standard.

## How to contribute

**Issues** — Open an issue to propose changes, report problems with the standard, or share how you're using `IDEA.md` in practice.

**Pull requests** — PRs are welcome for fixes, clarifications, and new examples. For changes to the core standard (`IDEA.standard.md`), please open an issue first to discuss.

**Examples** — The most useful contributions right now are real-world `IDEA.md` examples. If you've written one for a project, consider adding it to the `examples/` directory.

## Guidelines

- Keep the standard simple. The value of `IDEA.md` comes from what it leaves out.
- Resist adding required fields or sections. If something is useful but situational, make it optional.
- Write in plain language. The standard should be readable by anyone, not just developers.
- Test your changes by asking: would this make sense to an agent that has never seen the repo before?

## What doesn't belong here

- Tool-specific instructions (those go in `CLAUDE.md`, `GEMINI.md`, etc.)
- Implementation details or code
- Marketing language
